# Título da aula: Tratamento de Erros e Exceções

Tópicos abordados:
- Erros
- Tratamento de Exceções

O tratamento de erros não considera erros de sintaxe, ou seja, de digitação de comandos,
mas são erros de lógica. Para esses erros, damos o nome de Exceção.

Tratamento no Python se assemelha a outras linguagens (try, catch).

try:
    Operação sem falha
Except:
    Operação com falha

Exemplo: Divisão por Zero
    try:
        a = int(input('Numerador: '))
        b = int(input('Denominador: '))
        r = a / b
    except:
        print('Não foi possível realizar a tarefa.')
    else: # Cláusula opcional -> Só aparece se não tiver erro
        print(f'O resultado é {r}')
    finally: # Cláusula opcional -> Aparece no final de cada execução, ainda que tenha erro.
        print('Volte sempre! Muito Obrigado!')

Utilizar a exeção:
    try:
    [...]

    except Exception as erro:
        print(f'O erro encontrado foi {erro.__class__}') # Exibe a classe a qual o erro se refere.

Observação

Cada try pode ter vários except para cada uma das exceções:

    try:
    [...]

    except TypeError:
    [...]

    except ValueError:
    [...]
    
    except OSError:
    [...]

Exemplo: Adicionando várias exceções
    try:
        a = int(input('Numerador: '))
        b = int(input('Denominador: '))
        r = a / b

    except (ValueError, TypeError):
        print('Tivemos um problema com os tipos de dados que você digitou. Verifique os dados e tente novamente.')
    
    except ZeroDivisionError:
        print('Não é possível dividir nenhum número por zero!')
    
    except KeyboardInterrupt:
        print('O usuário preferiu não informar os dados.')

    except Exception as erro:
        print(f'O erro encontrado foi {erro.__cause__}')
